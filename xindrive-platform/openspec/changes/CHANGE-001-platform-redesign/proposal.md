# CHANGE-001: Platform Redesign — Level 4 Complete Rebuild

## 變更摘要

完全重新設計芯智驅 XinDrive AI 管理平台。從單一 1100 行 index.html 重建為模組化 SPA 架構，全新視覺語言、導航結構、故事化 Demo 資料，打造「讓客戶 5 分鐘內想買單」的產品展示體驗。

## 變更動機

### 現狀問題

1. **架構債務**：1106 行全塞一個 HTML 檔，無法維護與擴展
2. **視覺層級不足**：傳統管理後台外觀，無法傳達品牌價值與產品願景
3. **導航混亂**：所有功能平鋪在 Sidebar，角色不同但看到相同介面
4. **Demo 無說服力**：隨機填充的假資料，無法讓潛在客戶產生共鳴
5. **與商業模式錯位**：平台設計像後台工具，但商業模式需要「能賣」的產品體驗

### 商業背景

- 芯智驅定位為 IC 通路產業**獨立產品**（非顧問服務附屬工具）
- 三階段遞進商業模式：課程授權 → 教練服務 → 平台訂閱
- Q2 2026 需要 MVP Demo 版展示給潛在 Beta 客戶
- 已有客戶接洽中，需要可展示的產品原型

## 變更範圍

### 全面重建（Level 4）

| 層面 | 舊 | 新 |
|------|-----|-----|
| 架構 | 單一 index.html | 模組化 ES Modules SPA |
| 視覺 | 傳統管理後台 | 品牌級設計語言（三支柱色彩系統） |
| 導航 | 左側 Sidebar 平鋪 | Top Bar 三支柱切換 + 子導航 |
| Demo 資料 | seedDemoData() 隨機 | 故事化（芯達科技，3 角色切換） |
| 技術 | Vanilla JS 單檔 | Vanilla JS ES Modules 多檔案 |
| 後端 | localStorage | localStorage + Store 抽象層 |

### 不變的部分

- 產品定位：IC 產業 AI 驅動培訓・陪跑・業績倍增平台
- 三大支柱：驅・知（LEARN）、驅・伴（COACH）、驅・效（BOOST）
- 品牌色系基底：Navy #1B3A5C / Learn Blue #2563EB / Coach Green #10B981 / Boost Gold #F59E0B
- 零外部框架依賴原則
- GitHub Pages 靜態部署

## 成功標準

1. 全新平台可在 5 分鐘 Demo 流程中展示完整三支柱願景
2. 可切換 3 種角色（總監/業務/教練）看到差異化首頁
3. 每個支柱至少 2 個可互動頁面（非靜態假頁面）
4. 所有頁面 RWD 支援（桌面 + 平板 + 手機）
5. 檔案結構清晰，單一檔案不超過 200 行
6. Demo 資料講一個完整故事，客戶看完能理解價值
