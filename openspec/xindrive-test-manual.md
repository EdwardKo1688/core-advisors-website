## 3. 測試案例

### TC-01：登入功能

| 步驟 | 操作 | 預期結果 |
|------|------|---------|
| 1 | 開啟 http://localhost:8081 | 顯示登入頁面，含品牌標識、Email/密碼輸入框、3 個 Demo 角色卡 |
| 2 | 不輸入 Email，點「登入」 | 顯示紅色錯誤提示「請輸入電子郵件」 |
| 3 | 輸入 `lin@sinda.com`，密碼任意，點「登入」 | 跳轉至 Dashboard，顯示歡迎 Toast「歡迎回來，林志明」 |
| 4 | 輸入不存在的 Email，點「登入」 | 顯示錯誤提示「找不到此帳號，請使用 Demo 角色登入」 |
| 5 | 點擊「林總監」Demo 角色卡 | 直接跳轉至 Dashboard（Manager 視角） |
| 6 | 點擊「陳業務」Demo 角色卡 | 直接跳轉至 Dashboard（Sales 視角） |
| 7 | 點擊「張教練」Demo 角色卡 | 直接跳轉至 Dashboard（Trainer 視角） |

## 4. 資料重置

如需重置所有 Demo 資料回初始狀態：

```javascript
// 在瀏覽器 Console 執行
localStorage.removeItem('xindrive_data');
localStorage.removeItem('xindrive_user');
location.reload();
```

## 5. 已知限制

| 項目 | 說明 |
|------|------|
| 後端 API | 目前為 Demo 模式（localStorage），尚未接入 Supabase |
| AI 教練 | 對話為預設內容，尚未接入 LLM API |
| 資料持久性 | 清除瀏覽器資料會重置所有變更 |
| 密碼驗證 | Demo 模式不驗證密碼，任意輸入皆可登入 |
