# Proposal: 富昇物流 × 企業顧問專案管理系統

## Why（為什麼建立）

### 問題背景
1. **缺乏統一平台**：顧問在企業駐點時，對內管理（里程碑、Issue）、對外呈現（週報、文件）、對上簽核（審核意見）分散於 Excel、Email、Line 群組，造成資訊斷裂。
2. **無法追蹤閉環**：傳統 Excel 無法即時追蹤里程碑進度、KPI 達成率與問題閉環狀態，主管與審核者無法快速掌握專案全貌。
3. **知識不可攜帶**：三個月任務結束後，顧問的管理方法論與工具散落各處，無法系統性帶走複用。
4. **重複性文書工作**：每週撰寫週報、分析問題、追蹤 KPI 趨勢等工作耗時，AI 輔助可大幅提升效率。

### 預期效益
- 所有專案資訊集中一處，三角色各取所需
- 里程碑、Issue、KPI 全程追蹤，問題不漏接
- AI 輔助減少 60%+ 文書工作時間
- 三個月後完整移交富昇 IT，顧問帶走模板複用

## What（建立什麼）

### 六大業務模組
| # | 模組 | 核心功能 |
|---|------|----------|
| 01 | 專案總覽儀表板 | 進度概覽、圓餅圖、逾期警示、AI 智慧摘要 |
| 02 | 時程里程碑管理 | 甘特圖、CRUD、狀態自動判斷、進度百分比 |
| 03 | 每週進度週報 | 四欄位表單、AI 自動草稿、PDF 匯出、留言 |
| 04 | 問題與待辦追蹤 | Kanban 看板、拖曳、留言、AI 分析建議 |
| 05 | 交付文件管理 | 上傳/版本控制/預覽、AI 智慧搜尋 |
| 06 | KPI 成效儀表板 | 自訂指標、折線圖、達成率、AI 趨勢預測 |

### 四項 AI 輔助功能
| # | 功能 | 說明 |
|---|------|------|
| AI-01 | 週報自動摘要 | 分析本週所有模組活動，自動生成週報草稿 |
| AI-02 | 問題分析建議 | 分析 Issue 模式，建議優先順序與根因 |
| AI-03 | KPI 趨勢預測 | 根據歷史數據預測趨勢，標示風險 |
| AI-04 | 智慧文件搜尋 | 自然語言搜尋已上傳文件內容 |

### 系統層
- **角色權限**：Supabase Auth + RLS，三角色（顧問/主管/審核者）
- **系統日誌**：所有操作自動記錄，append-only，可查詢匯出
- **文件中心**：架構文件、變更紀錄、測試紀錄、系統日誌查詢、使用手冊
- **API 層**：Next.js API Routes + Swagger 文件

### 開發文件體系
- 架構文件（ARCHITECTURE.md）
- 變更紀錄（CHANGELOG.md）
- 使用手冊（USER_MANUAL.md）
- 架構決策紀錄（ADR）

## Stakeholders

| 角色 | 身份 | 系統權限 |
|------|------|----------|
| 顧問（我） | 管理員 | 完整 CRUD、觸發 AI 功能、管理 KPI、刪除文件 |
| 富昇主管 | 檢視者 | 唯讀瀏覽所有模組、留言、匯出 PDF/CSV |
| 老闆/審核者 | 審核者 | 唯讀瀏覽、留言、審核意見、電子簽核 |
| 富昇 IT | 接收者 | 三個月後接收完整程式碼與文件 |

## Scope

### In Scope
- Web 系統（Vercel 部署，對外可訪問）
- 六大業務模組完整功能
- 四項 AI 輔助功能（基於 Claude API）
- 三角色權限控制（RLS）
- 文件中心五個子頁面
- 系統日誌完整記錄
- 完整開發文件體系
- 響應式設計（桌機優先，手機可讀）

### Out of Scope
- 手機原生 App
- 即時通訊 / 聊天功能
- ERP / 會計系統整合
- 全自動 AI Agent（僅基礎輔助層級）
- 多專案切換（單一專案系統）

## 成功標準

1. 六大模組全部可操作（CRUD + 檢視 + 匯出）
2. 四項 AI 功能可正常生成結果且標記為「AI 生成」
3. 三角色權限正確隔離（RLS 測試通過）
4. 所有文件中心頁面可正常存取
5. 系統日誌完整記錄所有操作（含 AI 使用）
